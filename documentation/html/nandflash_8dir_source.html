00030 /**
00031  * \page nandflash NandFlash
00032  * 
00033  * \section Purpose
00034  * 
00035  *  This directory contains nandflash drivers
00036  * 
00037  * \section Description
00038  * NAND Flash driver package is composed of low-level functions for NAND Flash devices 
00039  * and external API functions that support the unified interface of functions and operations
00040  * for all supported NAND Flash Devices. The NAND Flash driver can be used for reference
00041  * code that demonstrates usage of low-level functions.
00042  * The NAND Flash drivers can be divided set of layers:
00043  *  - Raw Layer \n
00044  * The RawNandFlash layer drives hardware-specific Bus Read and Bus Write operations to 
00045  * communicate with the NAND Flash device. It directly interacts with hardware's register to 
00046  * operate NAND Flash interface.The RawNandFlash layer code implement procedures to program 
00047  * basic NAND Flash operations. It takes care of issuing the correct sequences of write/read 
00048  * operations for each command. All functions in the layer are blocked i.e. they wait for 
00049  * the completion of an operation. RawNandFlash layer access NAND Flash device by SMC.
00050  *  - Ecc Layer \n
00051  * Error Checking and Correction (ECC) to be calculated for each page of data. The
00052  * ECC data is used to detect when a memory location 'goes bad' and (depending on the ECC
00053  * algorithm) fix the bad bit. Typically, three bytes of ECC are generated for each page 
00054  * and placed in the 'spare area' of the page. The ECC algorithm in the NAND Flash driver 
00055  * support software Hamming ECC algorithm.
00056  *  - SkippedBlock Layer\n 
00057  * This layer is to just skip over the bad blocks and place the data in the known good blocks.
00058  * The algorithm starts by reading the entire spare area of the entire memory. The addresses 
00059  * of the factory-marked bad blocks are then collected in the programmer RAM. Next, the image is
00060  * sequentially programmed (page by page) into the target device. When the target address 
00061  * corresponds to a bad block address, these pages are stored in the next good block, skipping 
00062  * the bad block.
00063  * The SkipBlocks method is a very generic and well-performing strategy. When a bad block is
00064  * encountered, the algorithm simply skips ahead to the next good block.
00065  *  - Translated Layer\n
00066  * TranslatedNandFlash is an efficient layer to manage read and write access to flash.It adaptively
00067  * switches logical addresses to physical addresses in the flash memory by using block mapping,
00068  * page mapping technique.
00069  */
