00030 /**
00031  * \Page spi_dataflash SPI Dataflash Example
00032  *
00033  * \section Purpose
00034  *
00035  * The spi_dataflash example will help new users get familiar with SPI interface
00036  * on sam3s. This example gives you an AT45 Dataflash programming code so
00037  * that can help develop your own SPI devices applications with maximum
00038  * efficiency.
00039  *
00040  * You can find following information depends on your needs:
00041  * - A Spi low level driver performs SPI device Initializes, data transfer and
00042  * receive. It can be used by upper SPI driver such as AT45 %dataflash.
00043  * - A Dataflash driver is based on top of the corresponding Spi driver.
00044  * It allow user to do operations with %dataflash in a unified way.
00045  *
00046  * \section Requirements
00047  *
00048  * This package can be used with sam3s-ek and external Data Flash connected.
00049  * Please connect the SPI peripheral to external board like following matching
00050  * pairs:
00051  *        - <b>SAM3S--DataFlash</b>
00052  *        - VCC--VCC
00053  *        - GND--GND
00054  *        - NPCS0--NSS
00055  *        - MISO--MISO
00056  *        - MOSI--MOSI
00057  *        - SPCK--SPCK
00058  *
00059  * \section Description
00060  *
00061  * The demonstration program tests the dataflash connected to the evaluation kit by
00062  * erasing and writing each one of its pages.
00063  *
00064  * \section Requirements
00065  *
00066  * -# Build the program and download it inside the evaluation board. Please
00067  *    refer to the <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6224.pdf">SAM-BA User Guide</a>,
00068  *    the <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">GNU-Based Software Development</a>
00069  *    application note or to the <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">IAR EWARM User Guide</a>,
00070  *    depending on your chosen solution.
00071  * -# On the computer, open and configure a terminal application
00072  *    (e.g. HyperTerminal on Microsoft Windows) with these settings:
00073  *   - 115200 bauds
00074  *   - 8 bits of data
00075  *   - No parity
00076  *   - 1 stop bit
00077  *   - No flow control
00078  * -# Start the application.
00079  * -# Upon startup, the application will output the following lines on the terminal.
00080  *    \code
00081  *    -- SPI Dataflash Example xxx --
00082  *    -- xxxxxx-xx
00083  *    -- Compiled: xxx xx xxxx xx:xx:xx --

00087  *    \endcode
00088  * -# As soon as a dataflash is connected, the tests will start. Eventually,
