Get Started

The SAM3S is a Cortex-M3 based processor. This processor is designed to achieve high system performance in power and cost-sensitive embedded applications. It is also designed to be fast and easy to program, no assembler code or deep knowledge of the architecture to create simple application are required. The Getting Started example provides a simple template to start a new project on SAM3S.

CMSIS initiative

CMSIS defines for a Cortex-M3 Microcontroller System the register names of the Core Peripherals and the names of the Core Exception Vectors. The Getting Started example uses CMSIS definition. Atmel provides also the "Device Peripheral Access Layers" for SAM3S. The "how to benefit from Atmel header files V4" application note describes the organization of these definitions.

CM3 Core

CM3 core implements a new Hardware-based interrupt scheme. It allows deterministic and lower latency interrupts. The cm3 nvic example highlights the numerous optimization for nested and re-entrant interrupts and the priority management. Different from previous ARM architecture, atomic Bit Set or Clear can be performed through memory Bit Banding. The Bit Banding example shows how to benefit from this feature with simple macro.

Safety and Security

CM3 core implements two operating modes and two level of access. The mpu allows data used by privileged code (kernel) to be protected from untrusted user programs.
SAM3S peripheral WDT can be used to prevent system lock-up.

Microcontroller features

SAM3S Microcontroller has many peripheral controllers useful to interact directly with its environment.

  • ACC controls the Analog Comparator to compares two voltages and the result of this comparison. ACC IRQ Example shows how to compare two voltage inputs and and trigger it.
  • ADC can convert analog inputs to digital format. ADC12 Example shows how to configure an adc and use it with PDC channel to sample the voltage from a potentiometer. ADC12 Temperature Sensor Example gives an example that shows how to use the integrated Temperature Sensor through ADC channel 15. The ADC peripheral also features automatic comparison functions. How to program a comparison window to wake up the core in an appropriate manner is detailed in ADC12 Threshold Wakeup Example.
  • CRCCU integrates a dedicated Cyclic Redundancy Check (CRC) engine. CRCCU Example details the usage of CRCCU to compute CRC on a memory area.
  • DACC support two channels output which converts a digital code to analog value. DAC12 Sinewave Example gives an example of how to use the DACC to generate sinewave signals.
  • HSMCI supports the MultiMedia Card (MMC) Specification V4.3, the SD Memory Card Specification V2.0, the SDIO V2.0 specification and CE-ATA V1.1. The Basic SD/MMC Card Example gives deatils on how to implement SD drivers through HSMCI interface.
  • PIO Alternate Function Example gives an example how to alternate ERASE pin between system I/O line and PIO line. microcontroller has the function to alternate some system dedicated pin between PIOs, which extend PIO from existing general purpose I/O.
  • PIO Parallel Capture Example demonstrates how to read data from a CMOS digital image sensor, a high-speed parallel ADC, a DSP synchronous port in synchronous mode, etc.... using the PIO Controller.
  • PMC Clock Failure Detect Example details how to use the feature to detect a clock failure. PMC Clock Switching Example. details how to switch from a clock to another (PLLA, PLLB, SLCK, MAINCK) or change divider. PMC provides the Clock Failure Detector to monitor the 3 to 20 MHz Crystal or Ceramic Resonator based oscillator and to detect an eventual defect of this oscillator.
  • PWM can generate output waveforms and configure it's characteristics. PWM with PDC Example proves the PWM functions with PDC.
  • RTC provides date and time management as well as alarm configuration. RTC Example demonstrates how to configure the RTC in order to generate clock, alarm and calendar.
  • RTT is built around a 32-bit counter and used to count elapsed seconds. RTT Example demonstrates how to set and trigger an alarm by real-time timer.
  • The SMC is capable of handling several types of external memory and peripheral devices. How to configure SMC to access a LCD and drawing text, image, basic shapes on LCD is detailed in the SMC LCD Example.
  • SPI provides communication with external devices in Master or Slave Mode. Read and write external component by SPI is shown in the SPI Touchscreen Example which is also demonstrate how to use ADS7843 for touchscreen driver. While the SPI peripheral act as 'slaves', SPI Slave Example descripts how to configure SPI in slave mode to communicate to another SPI master interface.
  • SSC provides a synchronous communication link with external devices, SSC with PDC Audio Example describes the configuration required to connect the SSC to a device with an I2S-compatible serial bus.
  • The Timer Counter "TC" includes three identical 16-bit Timer Counter channels, Each channel can be independently programmed to perform frequency measurement, event counting, interval measurement, pulse generation. TC Capture Waveform Example indicate how to use TC in capture mode to measure the pulse width and count the total pulse number of an external signal injected on TIOA pin.
  • TWI interconnects components on a unique two-wire bus, TWI EEPROM Example and TWI Temperature Sensor Example demonstrates how to use the TWI peripheral to access an external serial EEPROM and a temperature sensor. TWI Slave Example demonstrates the TWI peripheral in slave mode. It mimics the behavior of a serial memory, enabling the TWI master to read and write data in its internal SRAM.
  • USART provides one full duplex universal synchronous asynchronous serial link. The usart synchronous mode usage is demonstrated in USART Synchronous Mode Example. The device is compliant with IrDA specification 1.1 and supports speed up to 115.2 Kb/s, a basic example of using USART in this mode is USART IRDA Example. The USART also features an ISO7816-compatible operating mode, the operation in this mode could be found at USART ISO7816 Example. The USART can works in SPI mode, USART SPI Mode Example would show the example of reading and writing as master or slave. The RS485 compliant mode is shown in the USART RS485 example, and hardware flow control implementation is illustrated in USART hardware handshaking Example.
  • UDP is compliant with the Universal Serial Bus (USB) V2.0 full-speed device specification. An example on how to implement USB Mass Storage class is detailed in USB Device Mass Storage Example. The USB CDC serial converter shows how to implement USB Communication Device class based on the UDP peripheral. USB Audio Speaker Example details the implementation on USB Audio Device class.
  • WDT can be used to prevent system lock-up if the software becomes trapped in a deadlock. Watchdog with IRQ Interrupt Example configures the watchdog and trigger a interrupt(or reset).

SAM3S memory access

SAM3S flash

CM3 Harvard architecture is optimized for single-cycle Flash usage. The EEFC programming example example details basic programming operations through SAM3S EEFC interface. The SAM3S embedded flash allows 128-bit or 64-bit access. You can access it through EFC peripheral. The Flash Memory Interface manages the programming, erasing, locking and unlocking sequences with dedicated commands. The Memory benchmark illustrates performance for different use cases. A 128-bit factory programmed unique ID may serve several purposes. The EEFC unique id example details how to generate an IPv6 address from this 128-bit value. To optimize power consumption, when Flash is accessed at low speed an automatic sleep mode is enabled. Benefits can be observed monitoring power consumption while running the Low power example. At least programming operations are forbidden while fetching code from the flash. The SAM3S ROM code embeds small In Application Programming Procedure. The EEFC programming example shows how to use these routines. The flash_applet serve as a monitor to implement flash program operations for SAM-BA.

Accessing memory through SPI

User could access on an external dataflash and serial flash component through SPI. SPI PDC driver performs SPI device Initializes, spi transfer and receive with PDC. SPI AT45 driver provides SPI interface with Dataflash and AT45 driver server common access operations such as read, write and erase. spi_dataflash help user get familiar with SPI interface on Atmel's AT45 dataflash family.

Accessing memory through SMC

Static Memory Controller (SMC) is capable of handling several types of external memory and peripheral devices, such as SRAM, PSRAM, PROM, EPROM, EEPROM, LCD Module, NOR Flash and NAND Flash.
The SMC Psram Example describes how to configure the Static Memory Controller for PSRAM, and do write and read operations to check the configuration. File-systems and USB mass-storage applications may use the "NAND FTL library" to perform safe access to the NAND in term of power safe, wear leveling and bad block management. The SMC NAND Flash Example show how to read and write data from/to a nandflash connected to the SMC, taking ECC and Bad Block marking into account.The FATFS Nandflash Example will give details to implement filesystem through a NAND FLASH based memory. SMC NOR Flash Example gives you CFI supported Norflash programming through the External Bus Interfac (EBI).

Memories library

Memories library is composed of NorFlash, NandFlash and SD/MMC Library low lever drivers. This library gives user some reference to develop NAND flash, NOR flash and SD card applications with more efficiency.
