00030 /**
00031  *  \page pio_capture PIO Parallel Capture Example
00032  *
00033  *  \section Purpose
00034  *
00035  *  The PIO Parallel Capture demonstrates how to read data from a CMOS digital image
00036  *  sensor, a high-speed parallel ADC, a DSP synchronous port in synchronous
00037  *  mode, etc.... using the PIO Controller.
00038  *
00039  *  \section Requirements
00040  *
00041  *  This package can only be used with sam3s-ek.
00042  *
00043  *  \section Description
00044  *
00045  *  The PIO Controller integrates an interface able to read data from a CMOS
00046  *  digital image sensor, a high-speed parallel ADC, a DSP synchronous port in
00047  *  synchronous mode, etc....
00048  *
00049  *  The application is ocmposed of 2 softwares:
00050  *  - one for use the PIO Parallel Capture in send mode.
00051  *  - one for use the PIO Parallel Capture in receive mode.
00052  *
00053  *  The use of 2 boards is required. Connect one to the other, and put one board
00054  *  in send mode and the second in receive mode.
00055  *  Different choice can be selected for use the data enable pins, or not, and
00056  *  for sample all data, or only one time out of two.
00057  *  Pins to be connected beetween the 2 boards:<br />
00058  *    PA15 PIODCEN1<br />
00059  *    PA16 PIODCEN2<br />
00060  *    PA17 AD0<br />
00061  *    PA18 AD1<br />
00062  *    PA19 AD2/WKUP9<br />
00063  *    PA20 AD3/WKUP10<br />
00064  *    PA21 AD8<br />
00065  *    PA22 AD9<br />
00066  *    PA23 PIODCCLK<br />
00067  *    PA24 PIODC0<br />
00068  *    PA25 PIODC1<br />
00069  *    PA26 PIODC2<br />
00070  *    PA27 PIODC3<br />
00071  *    PA28 PIODC4<br />
00072  *    PA29 PIODC5<br />
00073  *    PA30 PIODC6<br />
00074  *    PA31 PIODC7<br />
00075  *    And, of course: GND<br />
00076  *
00077  *  \section Usage
00078  *
00079  *  -# Build the program and download it inside the evaluation board. Please
00080  *     refer to the
00081  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6224.pdf">
00082  *     SAM-BA User Guide</a>, the
00083  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">
00084  *     GNU-Based Software Development</a>
00085  *     application note or to the
00086  *     <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">
00087  *     IAR EWARM User Guide</a>,
00088  *     depending on your chosen solution.
00089  *  -# On the computer, open and configure a terminal application
00090  *     (e.g. HyperTerminal on Microsoft Windows) with these settings:
00091  *    - 115200 bauds
00092  *    - 8 bits of data
00093  *    - No parity
00094  *    - 1 stop bit
00095  *    - No flow control
00096  *  -# Connect the first board to the second board by connecting:
00097  *     PA15, PA16, PA23, PA24, PA25, PA26, PA27, PA28, PA29, PA30, PA31 and GND.
00098  *  -# Start application of the first board.
00099  *  -# In the terminal window, the
00100  *     following text should appear (values depend on the board and chip used):
00101  *     \code
00102  *      -- PIO Parallel Capture example x.x --
00103  *      -- SAM3S-EK
00104  *      -- Compiled: xxx xx xxxx xx:xx:xx --
00105  *      Frequency: 64 MHz
00106  *        Press r to Receive data on PPIO Parallel Capture
00107  *        Press s to Send data on PIO Parallel Capture
00108  *      ** SEND mode **
00109  *      This is for debug purpose only !
00110  *      Frequency of PIO controller clock must be strictly superior to 2 times
00111  *      the frequency of the clock of the device which generates the parallel data.
00112  *
00113  *      Please, connect the second board, and put it in reception mode
00114  *        Press y to send data without data enables pins
00115  *        Press n to send data with data enables pins
00116  *      We send data without data enables pins
00117  *        Press a key

00120  *     \endcode

00128  *      -- Compiled: xxx xx xxxx xx:xx:xx --
00129  *      Frequency: 64 MHz
00130  *        Press r to Receive data on PIO Parallel Capture

00132  *      ** RECEIVE mode **
00133  *        Press y to samples the data when both data enables are active
00134  *        Press n to samples the data whatever the data enables are
00135  *      We receive data whatever the data enables are
00136  *        Press y to samples all the data
