00030 /**
00031  *  \page usart_rs485 USART RS485 Example
00032  *
00033  *  \section Purpose
00034  *
00035  *  The USART RS485 Example demonstrates how to use USART in RS485 mode.
00036  *
00037  *  \section Requirements
00038  *
00039  *  This package can only be used with sam3s-ek. Before running,
00040  *  make sure to connect two boards with RS485 lines.The connector J4 is for
00041  *  this purpose. Match each paired pins of two boards respectively with A to A,
00042  *  B to B and FGND to FGND(the central pin of J4).
00043  *
00044  *  The R25 (0R) is not populated for conflicting between RS232 and RS485. Make
00045  *  sure to populate it properly before running this example.
00046  *
00047  *  \section Description
00048  *
00049  *  This example connects two boards through RS485 interface. One board acts
00050  *  as the transmitter and the other one as the receiver. It is determined by
00051  *  the sequence the two applications started.
00052  *
00053  *
00054  *  In any case, the application sends a sync character at running to seek an
00055  *  receiver.If the acknowledgment is received in a short time, it will act
00056  *  as the transmitter and then send a full frame text to the receiver.
00057  *
00058  *
00059  *  The earlier started board will act automatically as the receiver due to no
00060  *  acknowledgment received. The receiver will wait until sync character is
00061  *  received. Then it sends the acknowledgment and waits for the full frame
00062  *  sent by the transmitter. At the end of reception,it prints out message
00063  *  through UART interface to assert that the whole process succeeds.
00064  *
00065  *  \section Usage
00066  *
00067  *  -# Build the program and download it inside the evaluation board. Please
00068  *     refer to the
00069  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6224.pdf">
00070  *     SAM-BA User Guide</a>, the
00071  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">
00072  *     GNU-Based Software Development</a>
00073  *     application note or to the
00074  *     <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">
00075  *     IAR EWARM User Guide</a>,
00076  *     depending on your chosen solution.
00077  *  -# On the computer, open and configure a terminal application for each board
00078  *     (e.g. HyperTerminal on Microsoft Windows) with these settings:
00079  *    - 115200 bauds
00080  *    - 8 bits of data
00081  *    - No parity
00082  *    - 1 stop bit
00083  *    - No flow control
00084  *  -# Start application from two boards in sequence. Make sure the second board
00085  *     should NOT be started unless the first board had run to wait for the
00086  *     synchronizing character. The output message in later section would
00087  *     describe this.
00088  *
00089  *  -# In the terminal  window, the
00090  *     following text should appear (values depend on the board and chip used):
00091  *     \code
00092  *      -- USART RS485 Example xxx --
00093  *      -- xxxxxx-xx
00094  *      -- Compiled: xxx xx xxxx xx:xx:xx --
00095  *     \endcode
00096  *  -# The consequent messages will indicate the boards' behavior.
00097  *
00098  *     -  The earlier started board would output the message below to indicate it
00099  *     was waiting for a synchronization character:
00100  *     \code
00101  *     -I- Receiving sync character.
00102  *     \endcode
00103  *     -  If received a sync character and prepared to receive a frame,it
00104  *     printed out the message below:
00105  *     \code
00106  *     -I- Start receiving!
00107  *     \endcode
00108  *     -  After successfully received a frame, the board output the following
00109  *     message to indicate that the whole process succeeded.
00110  *     \code
00111  *     -I- Received successfully!
00112  *     \endcode
00113  *     -  The later started one would act as transmitter,and if received an
00114  *     acknowledgment character successfully, it output the following message
00115  *     and started transmitting:
00116  *     \code
00117  *     -I- Start transmitting!
00118  *     \endcode
00119  *
00120  *  \section References
