00008 \section get_started_sec Get Started
00009 The SAM3S is a Cortex-M3 based processor. This processor is designed to achieve high system performance in power and cost-sensitive embedded applications.  It is also designed to be fast and easy to program, no assembler code or deep knowledge of the architecture to create simple application are required. The \ref getting_started "Getting Started" example provides a simple template to start a new project on SAM3S.

00015 \section cm3_core_sec CM3 Core
00016 CM3 core implements a new Hardware-based interrupt scheme. It allows deterministic and lower latency interrupts. The \ref cm3_nvic "cm3 nvic" example highlights the numerous optimization for nested and re-entrant interrupts and the priority management.
00017 Different from previous ARM architecture, atomic Bit Set or Clear can be performed through memory Bit Banding. The \ref cm3_bit_banding "Bit Banding example" shows how to benefit from this feature with simple macro.

00023 \section microcontroller_sec Microcontroller features
00024 SAM3S Microcontroller has many peripheral controllers useful to interact directly with its environment.\n
00025 - \ref acc_module "ACC" controls the Analog Comparator to compares two voltages and the result of this comparison. \ref acc_irq "ACC IRQ Example" shows how to compare two voltage inputs and and trigger it.
00026 - \ref adc_module "ADC" can convert analog inputs to digital format. \ref adc12 shows how to configure an adc and use it with PDC channel to sample the voltage from a potentiometer. \ref adc12_temp_sensor gives an example that shows how to use the integrated Temperature Sensor through ADC channel 15.
00027 The ADC peripheral also features automatic comparison functions. How to program a comparison window to wake up the core in an appropriate manner is detailed in \ref adc12_threshold_wakeup.\n
00028 - \ref crccu_module "CRCCU" integrates a dedicated Cyclic Redundancy Check (CRC) engine. \ref crccu "CRCCU Example" details the usage of CRCCU to compute CRC on a memory area.
00029 - \ref dacc_module "DACC" support two channels output which converts a digital code to analog value. \ref dac12_sinewave gives an example of how to use the DACC to generate sinewave signals.\n
00030 - \ref hsmci_module "HSMCI"  supports the MultiMedia Card (MMC) Specification V4.3, the SD Memory Card Specification V2.0, the SDIO V2.0 specification and CE-ATA V1.1. The \ref hsmci_sdcard "Basic SD/MMC Card Example"  gives deatils on how to implement SD drivers through HSMCI interface.
00031 - \ref pio_alternate_function "PIO Alternate Function Example" gives an example how to alternate ERASE pin between system I/O line and PIO line.\nThe microcontroller has the function to alternate some system dedicated pin between PIOs, which extend PIO from existing general purpose I/O.
00032 - \ref pio_capture_module "PIO Parallel Capture Example" demonstrates how to read data from a CMOS digital image sensor, a high-speed parallel ADC, a DSP synchronous port in synchronous mode, etc.... using the PIO Controller.
00033 - \ref pmc_clock_failure_detect "PMC Clock Failure Detect Example" details how to use the feature to detect a clock failure. \ref pmc_clock_switching details how to switch from a clock to another (PLLA, PLLB, SLCK, MAINCK) or change divider.\nThe PMC provides the Clock Failure Detector to monitor the 3 to 20 MHz Crystal or Ceramic Resonator based oscillator and to detect an eventual defect of this oscillator.
00034 - \ref pwm_module "PWM" can generate output waveforms and configure it's characteristics. \ref pwm_pdc "PWM with PDC Example" proves the PWM functions with PDC.
00035 - \ref rtc_module "RTC" provides date and time management as well as alarm configuration. \ref rtc demonstrates how to configure the RTC in order to generate clock, alarm and calendar.\n
00036 - \ref rtt_module "RTT" is built around a 32-bit counter and used to count elapsed seconds. \ref rtt "RTT Example" demonstrates how to set and trigger an alarm by real-time timer.
00037 - The SMC is capable of handling several types of external memory and peripheral devices. How to configure SMC to access a LCD and drawing text, image, basic shapes on LCD is detailed in the \ref smc_lcd.\n
00038 - \ref spi_module "SPI" provides communication with external devices in Master or Slave Mode. Read and write external component by SPI is shown in the \ref spi_touchscreen which is also demonstrate how to use ADS7843 for touchscreen driver.
00039 While the SPI peripheral act as 'slaves', \ref spi_slave descripts how to configure SPI in slave mode to communicate to another SPI master interface.\n
00040 - \ref ssc_module "SSC" provides a synchronous communication link with external devices, \ref ssc_pdc_audio describes the configuration required to connect the SSC to a device with an I2S-compatible serial bus.
00041 - The Timer Counter "TC" includes three identical 16-bit Timer Counter channels, Each channel can be independently programmed to perform
00042 frequency measurement, event counting, interval measurement, pulse generation. \ref tc_capture_waveform indicate how to use TC in capture mode to measure the pulse width and count the total pulse number of an external signal injected on TIOA pin.
00043 - \ref twi_module "TWI" interconnects components on a unique two-wire bus, \ref twi_eeprom and \ref twi_temp_sensor demonstrates how to use the TWI peripheral to access an external serial EEPROM and a temperature sensor.
00044 \ref twi_slave demonstrates the TWI peripheral in slave mode. It mimics the behavior of a serial memory, enabling the TWI master to read and write data in its internal SRAM.
00045 - \ref usart_module "USART" provides one full duplex universal synchronous asynchronous serial link. The usart synchronous mode usage is demonstrated in \ref usart_synchronous. The device is compliant with
00046  IrDA specification 1.1 and supports speed up to 115.2 Kb/s, a basic example of using USART in this mode is \ref usart_irda. The USART also features an ISO7816-compatible operating mode, the operation in this mode
00047  could be found at \ref usart_iso7816. The USART can works in SPI mode, \ref usart_spi would show the example of reading and writing as master or slave.
00048 The RS485 compliant mode is shown in the \ref usart_rs485 "USART RS485 example", and hardware flow control implementation is illustrated in \ref usart_hw_handshaking "USART hardware handshaking Example".
00049 - \ref usb_lib "UDP" is compliant with the Universal Serial Bus (USB) V2.0 full-speed device specification. An example on how to implement USB Mass Storage class is detailed in \ref usb_massstorage "USB Device Mass Storage Example". The \ref usb_cdc_serial "USB CDC serial converter" shows how to
00050    implement USB Communication Device class based on the UDP peripheral. \ref usb_audio_speaker "USB Audio Speaker Example" details the implementation on USB Audio Device class.
00051 - \ref wdt_module "WDT" can be used to prevent system lock-up if the software becomes trapped in a deadlock. \ref wdg_irq "Watchdog with IRQ Interrupt Example" configures the watchdog and trigger a interrupt(or reset).

00057 \section sam3s_memories_sec SAM3S memory access
00058     \subsection sam3s_flash_sec SAM3S flash
00059 CM3 Harvard architecture is optimized for single-cycle Flash usage. The \ref eefc_pgm example details basic programming operations through SAM3S EEFC interface.
00060 The SAM3S embedded flash allows 128-bit or 64-bit access. You can access it through \ref efc_module "EFC" peripheral. The \ref flashd_module manages the programming, erasing, locking and unlocking sequences with dedicated commands.
00061 The \ref mem_bench "Memory benchmark" illustrates performance for different use cases.
00062 A 128-bit factory programmed unique ID may serve several purposes. The \ref eefc_uniqueid details how to generate an IPv6 address from this 128-bit value.
00063 To optimize power consumption, when Flash is accessed at low speed an automatic sleep mode is enabled. Benefits can be observed monitoring power consumption while running the \ref lowpower "Low power example".
00064 At least programming operations are forbidden while fetching code from the flash. The SAM3S ROM code embeds small \ref iap_appnote "In Application Programming Procedure".
00065 The \ref eefc_pgm_example "EEFC programming example" shows how to use these routines.
00066 The \ref flash_applet serve as a monitor to implement flash program operations for SAM-BA.

00073     \subsection smc_sec Accessing memory through SMC
00074 Static Memory Controller (SMC) is capable of handling several types of external memory and peripheral devices, such as SRAM, PSRAM, PROM, EPROM, EEPROM, LCD Module, NOR Flash and NAND Flash.\n
00075 The \ref smc_psram describes how to configure the Static Memory Controller for PSRAM, and do write and read operations to check the configuration.
00076 File-systems and USB mass-storage applications may use the "NAND FTL library" to perform safe access to the NAND in term of power safe, wear leveling and bad block management.
00077 The \ref smc_nandflash show how to read and write data from/to a nandflash connected to the SMC, taking ECC and Bad Block marking into account.The \ref fatfs_nandflash "FATFS Nandflash Example"
00078 will give details to implement filesystem through a NAND FLASH based memory.
00079 \ref smc_norflash gives you CFI supported Norflash programming through the External Bus Interfac (EBI).
