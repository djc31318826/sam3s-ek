00031 /**
00032  *  \page pmc_clock_failure_detect PMC Clock Failure Detect Example
00033  *
00034  *  \section Purpose
00035  *
00036  *  This example shows how to use the Clock Failure Detection feature of the
00037  *  SAM3S chip when the external oscillator fails.
00038  *
00039  *  \section Requirements
00040  *
00041  *  This package can only be used with sam3s-ek.
00042  *
00043  *  \section Description
00044  *   The example requires the chip configured to run on external oscillator,
00045  *   which is normally the case when using the standard lowlevel init functions. *
00046  *   When the external oscillator is activated, the XIN pin is automatically set
00047  *   in XIN mode, and no more in PIO mode.
00048  *
00049  *   After startup,it enables Clock Failure Detector Enable feature
00050  *   (CFDEN in PMC_MOR register),and also the interrupt
00051  *   of Clock Failure Detector Event(CFDEV through PMC_IER).Then ask the user to
00052  *   force a oscillator failure by shorting XIN or XOUT pin to ground.The chip
00053  *   switches automatically MAINCK on the 4/8/12 MHz Fast RC Oscillator clock
00054  *   and also trigger the CFDEV interrupt.
00055  *
00056  *   In the interrupt handler, the example lighting a LED ON to show this
00057  *   feature works.
00058  *
00059  *  \section Usage
00060  *
00061  *  -# Build the program and download it inside the evaluation board. Please
00062  *     refer to the
00063  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6224.pdf">
00064  *     SAM-BA User Guide</a>, the
00065  *     <a href="http://www.atmel.com/dyn/resources/prod_documents/doc6310.pdf">
00066  *     GNU-Based Software Development</a>
00067  *     application note or to the
00068  *     <a href="ftp://ftp.iar.se/WWWfiles/arm/Guides/EWARM_UserGuide.ENU.pdf">
00069  *     IAR EWARM User Guide</a>,
00070  *     depending on your chosen solution.
00071  *  -# On the computer, open and configure a terminal application for each board
00072  *     (e.g. HyperTerminal on Microsoft Windows) with these settings:
00073  *    - 115200 bauds
00074  *    - 8 bits of data
00075  *    - No parity
00076  *    - 1 stop bit
00077  *    - No flow control
00078  *  -# Start application from two boards in sequence.
00079  *  -# In the terminal window, the
00080  *     following text should appear (values depend on the board and chip used):
00081  *     \code
00082  *      -- PMC Clock Failure Detect Example xxx --
00083  *      -- xxxxxx-xx
00084  *      -- Compiled: xxx xx xxxx xx:xx:xx --
00085  *     \endcode
00086  *  -# The consequent output message through UART should be:
00087  *     \code
00088  *     -I- Short XIN or XOUT to ground to force a clock failure.
00089  *     \endcode
00090  *     The user should short XIN or XOUT (97 and 96 of 100-Lead LQFP Pinout) to
00091  *     ground to force a clock failure. If the failure is detected, the LED D2
00092  *     would be lighted on in the event handler.
00093  *  \section References
00094  *  - pmc_clock_failure_detect/main.c
00095  *  - pmc.h
00096  */
